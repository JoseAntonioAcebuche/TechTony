# 📖 Overview

**PowerShell** is Microsoft's task automation and configuration management framework.

It combines a command-line shell with a powerful scripting language, allowing Database Administrators (DBAs) and System Administrators to automate repetitive SQL Server tasks.

PowerShell can manage:

- SQL Server Instances
- Databases
- SQL Server Agent Jobs
- Backups and Restores
- Always On Availability Groups
- Log Shipping
- Replication
- Windows Services
- Scheduled Tasks

PowerShell is an essential skill for modern SQL Server administrators.

# Why Use PowerShell?

Benefits include:

- Automates repetitive tasks
- Saves administrative time
- Reduces human error
- Supports remote administration
- Integrates with Windows Server
- Works with SQL Server Management Objects (SMO)

# Install SQL Server PowerShell Module

Install the official SQL Server module.

```powershell
Install-Module SqlServer -Scope CurrentUser
```

Verify installation.

```powershell
Get-Module SqlServer -ListAvailable
```

Import the module.

```powershell
Import-Module SqlServer
```

---

# Connect to SQL Server

```powershell
Invoke-Sqlcmd `
-ServerInstance "SQLSERVER01" `
-Database "master" `
-Query "SELECT @@VERSION"
```

---

# Execute a SQL Query

```powershell
Invoke-Sqlcmd `
-ServerInstance "SQLSERVER01" `
-Database "ParkingDB" `
-Query "SELECT * FROM dbo.Vehicle"
```

---

# Backup a Database

```powershell
Backup-SqlDatabase `
-ServerInstance "SQLSERVER01" `
-Database "ParkingDB" `
-BackupFile "D:\Backup\ParkingDB.bak"
```

---

# Restore a Database

```powershell
Restore-SqlDatabase `
-ServerInstance "SQLSERVER01" `
-Database "ParkingDB" `
-BackupFile "D:\Backup\ParkingDB.bak"
```

# Execute SQL Server Agent Job

```powershell
Invoke-Sqlcmd `
-ServerInstance "SQLSERVER01" `
-Query "
EXEC msdb.dbo.sp_start_job
@job_name='Nightly Backup'
"
```

---

# Export Query Results to CSV

```powershell
Invoke-Sqlcmd `
-ServerInstance "SQLSERVER01" `
-Query "SELECT * FROM dbo.Vehicle" |
Export-Csv D:\Reports\VehicleReport.csv -NoTypeInformation
```

---

# Generate HTML Report

```powershell
Invoke-Sqlcmd `
-ServerInstance "SQLSERVER01" `
-Query "SELECT * FROM sys.databases" |
ConvertTo-Html |
Out-File D:\Reports\DatabaseReport.html
```

---

# Limitations

- Requires PowerShell knowledge
- Module compatibility varies by SQL Server version
- Administrative privileges may be required
- Poorly written scripts can affect production systems

---

# Best Practices

- Store scripts in source control.
- Use descriptive script names.
- Test scripts in a development environment.
- Log script output.
- Use error handling with `try` and `catch`.
- Avoid storing passwords in plain text.
- Schedule long-running jobs during maintenance windows.

---

# Common Mistakes

❌ Running scripts directly in production without testing.

❌ Hardcoding credentials.

❌ Ignoring error handling.

❌ Not logging execution results.

❌ Running scripts with excessive privileges.
