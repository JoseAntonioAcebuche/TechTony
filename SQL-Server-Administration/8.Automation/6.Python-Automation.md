# Connect to SQL Server

```python
import pyodbc

connection = pyodbc.connect(
    "DRIVER={ODBC Driver 18 for SQL Server};"
    "SERVER=SQLSERVER01;"
    "DATABASE=ParkingDB;"
    "Trusted_Connection=yes;"
    "TrustServerCertificate=yes;"
)

cursor = connection.cursor()
```

---

# Execute a Query

```python
cursor.execute("SELECT @@VERSION")

for row in cursor.fetchall():
    print(row)
```

---

# Read Data with Pandas

```python
import pandas as pd

query = "SELECT * FROM dbo.Vehicle"

df = pd.read_sql(query, connection)

print(df.head())
```

# Backup Database

```python
backup_sql = """
BACKUP DATABASE ParkingDB
TO DISK='D:\\Backup\\ParkingDB.bak'
WITH INIT
"""

cursor.execute(backup_sql)
connection.commit()
```

# Schedule Tasks

```python
import schedule
import time

def backup():

    print("Running Backup")

schedule.every().day.at("23:00").do(backup)

while True:

    schedule.run_pending()

    time.sleep(1)
```

# Best Practices

- Store database credentials securely.
- Use environment variables.
- Implement proper exception handling.
- Use logging for every automation task.
- Close database connections properly.
- Test scripts before production deployment.
- Store scripts in Git.

---

# Common Mistakes

❌ Hardcoding passwords.

❌ Ignoring exceptions.

❌ Forgetting to close database connections.

❌ Running production scripts without testing.

❌ Not logging execution results.
